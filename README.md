# gitflow-example

1. Создать репо на github и склонировать его на компьютер.
2. Создать ветку разработки от главной и переключиться на неё.
   (После начинается процесс разработки)
3. От ветки develop создаются feature-ветки (main-page, about-page, modals, global-styles), с определённым функционалом
   После заверщения работы на feature-веткой она мержится в develop.
4. Создание ветки release
   (Данная ветка создаётся, когда приближается дата релиза
   через слэш присваивается версия release/0.1.0 данная ветка создаётся от develop
   после того, как была создана ветка release, добавлять новые фичи нельза, допускается только исправление багов
   .)
5. Когда ветка release/x.y.z закончена,то она мержится в develop и main и затем удаляетсяю
